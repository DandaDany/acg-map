## 後台模組對應

- 內容管理：`data/manual/`
- 資料管線：`backend/update_all.py`、`backend/refresh_venues.py`、`backend/collect_*.py`
- 圖像資產：`public/logos/` 與 `data/logos/`
- 地圖發布：`public/`
- 維護報表：`backend/report_status.py` 與 `data/reports/`
- 排程與工作狀態：`ops/` 與 `runtime/`

## 路徑規則

- 後台程式不可再寫死根目錄檔名，請使用 `backend/paths.py` 的 `P("檔名")`。
- `venues.json` 裡的 logo 值維持 `logos/...`，因為這是前端公開路徑；實體檔案位於 `public/logos/...`。
- `data/manual/` 視為人工資料與後台管理表單的未來來源；自動管線不應無故覆蓋人工修正。
- `data/generated/`、`data/cache/`、`runtime/` 可由程式重建或更新。
- 瀏覽器 profile 內部子資料夾由 Chrome/Playwright 自動管理，不逐層加入專案說明檔；說明集中放在 `runtime/profiles/` 與其第一層 profile 目錄。

## 常用指令

```bash
# 更新資料
python3 backend/update_all.py

# 本機預覽
cd public
python3 -m http.server 8000
# 開 http://localhost:8000/taiwan-exhibition-map.html

# 檢查核心程式與資料
python3 -m py_compile backend/update_all.py backend/refresh_venues.py backend/collect_logos.py
python3 -c 'import json; json.load(open("public/venues.json", encoding="utf-8")); print("venues json ok")'
```
