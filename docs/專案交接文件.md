## 1. 專案是什麼

一個**公開的單頁網頁地圖**，把全台「展覽／ACG 活動地點」標在地圖上：

- 每個場館一個圓形圖釘（官方 logo 或預設圖示），右上角徽章顯示該館目前展覽數。
- 點圖釘 → 底部滑出圖卡列表（主視覺 KV／名稱／日期／分類／**詳細地址**／官方連結）。
- 篩選：時間（全部／展覽中／即將開展／即將結束）× 類型（全部／特展／常設）× 分類（視覺藝術／歷史人文／科學自然／工藝設計／親子兒童／**動漫遊戲(ACG)**／其他）；另有關鍵字搜尋與北中南東跳轉。

技術上是**純靜態網頁**：`taiwan-exhibition-map.html` 載入時 `fetch('venues.json')`；本機雙擊（file://）則用 HTML 內嵌的備援快照（離線也能開，但無街道圖磚）。街道底圖與路名需連網。

## 3. 資料來源（四層）

### 第一層：文化部開放資料（政府 API，廣度）
- 資料集：政府資料開放平臺「藝文活動-所有類別」(dataset 6478)；本專案只取**展覽** `category=6`。
  - 資料集頁：`https://data.gov.tw/dataset/6478`
  - JSON API：`https://cloud.culture.tw/frontsite/trans/SearchShowAction.do?method=doFindTypeJ&category=6`
- 授權：政府資料開放授權條款-第1版；每日更新。
- **已知限制**：座標只在 `showInfo[].latitude/longitude`，且只有約 1/4 展覽有座標；`imageUrl` 幾乎都空；`locationName` 偶有「○○區（縣市）=」雜訊。→ 這是「區級定位」「官網補圖」「名稱用 masterUnit 補」的原因。`src=moc`。

### 第二層：各館官網爬蟲（深度，Playwright）
- `collect_venues.py`：通用 path/auto 模式，含**自動翻頁**。已納入華山、松菸、駁二、故宮、傳藝、國美館、纖維工藝、港區、嘉義文創、奇美、花蓮文創、臺南市美術館、蕭壠等。
- `collect_public.py`：結構特殊、需客製解析的公共場館：
  - **新竹市美術館**：新竹市文化局「活動資訊」div 列表，**民國年日期**（115=2026），用 `[展覽]` 標籤過濾。
  - **關渡美術館**：當期展覽卡片（換展期間會無資料，屆時自動回空）。
  - **C-LAB**：`/events/?event_category=exhibition`（只列展覽），日期格式特殊（`06.06 (六) 2026`）。
- 兩者皆輸出 `venue_extra.json`，`src=official`。

### 第三層：編輯整理（ANCHORS，在 `refresh_venues.py`）
- 重點公共館＋常設層，含已驗證官網與 OSM 精確座標。故宮、北美館、台博、MoNTUE、華山、松菸、國美館、文資園區、高美館、臺南市美術館、奇美、嘉義文創、花蓮文創，以及本次新增的 **C-LAB、關渡美術館、新竹市美術館、蕭壠文化園區、台北當代藝術館、鳳甲美術館**。`src=curated`。
- 註：若該 anchor 有對應官網爬蟲資料（venue_extra），會被取代為 `official`；若有當期展覽，則顯示真實展覽而非「詳見官網」卡。

### 第四層：使用者手動（manual_extra.json，ACG）
- 使用者提供的 ACG 活動 Excel（快閃店/展覽/物販），整理成 `manual_extra.json`。
- refresh 於**所有篩選之後**併入（不被 keep/砍歷史移除），分類一律「動漫遊戲(ACG)」，`src=manual`。
- **持久層**：每週自動更新會重新併入，故手動資料不會被洗掉；`endDate < 今天` 會自動略過（過期下架）。

### 地圖底圖
- **街道向量底圖**：MapLibre GL（OpenFreeMap Positron），透過 `leaflet-maplibre-gl` 外掛接到 Leaflet。道路在載入後改為加粗白線。需連網（CDN：unpkg + tiles.openfreemap.org）。
- 鄉鎮界 GeoJSON（g0v `twTown1982`，簡化內嵌）保留為淡色輔助框線（透明填滿）。
- 縣市/區地名標籤由前端用 `town_centroids` 自畫（遠→縣市、近→區）；路名由底圖圖磚於高縮放自動出現。

## 6. refresh_venues.py 處理步驟

1. **SSL**：開頭建立放寬 `VERIFY_X509_STRICT` 的 SSL context（保留信任鏈/主機名驗證），解決新版 OpenSSL 對部分政府憑證（缺 Subject Key Identifier）的拒絕。
2. 抓 `category=6` JSON；篩當期（`endDate >= 今天`）。
3. 定位：有合法座標→exact；否則解析地址縣市+鄉鎮區→`town_centroids` 區級；只到縣市→市級。同區依名稱雜湊微抖動。
4. 場館命名：用 `locationName`；雜訊值改用 `masterUnit`（`venue_name()`）。
5. **存來源地址**：把文化部 `location` 完整地址存成 `addr`（供圖卡顯示「縣市+區+路段」）。
6. 聚合：exact 依座標聚合；approx 依（縣市,名稱）聚合。
7. 併入 `ANCHORS`（`near()` 同點合併門檻 **0.003°**≈330m，避免把相距 1km 的不同館誤併）。
8. 併入 `venue_extra.json`（官網收集）→ 取代占位卡，`src=official`。
9. **排除**圖書館／郵政（`LIB_V`/`LIB_EX`）。
10. 圖片 KV 驗證（og:image 等，嚴格驗證，寧缺勿錯）。
11. 收斂刪除（`keep_venue`：official/curated/manual 與展館類名稱保留；行政/百貨/車站等非展場剔除）→ 名稱正規化合併（`canon`）→ 砍歷史類。
12. **分類**（7 類，混合法）：`CAT_TITLE`（含 ACG 關鍵字與常見 IP）優先 → `CAT_VENUE` → 其他/綜合。
13. **併入 `manual_extra.json`**（置於所有篩選之後，不被移除；強制分類 ACG；`src=manual`；過期略過）。
14. 輸出 `venues.json` 並印統計。

## 7. 前端 taiwan-exhibition-map.html

- **底圖**：`L.maplibreGL({style: OpenFreeMap Positron})`（CDN：`maplibre-gl` + `@maplibre/maplibre-gl-leaflet`）。載入後 `boldRoads()` 把主要道路改粗白線。鄉鎮界 GeoJSON 設透明填滿、淡框線。
- **地名標籤**：用內嵌 `TOWN_C`（鄉鎮中心座標）自畫；`syncLabels()` 於 zoom≥11 顯示「區」、否則顯示「縣市」。路名由底圖圖磚提供。
- **圖卡地址**：`venueAddr(v)` = `v.addr`（來源完整地址）→ 否則 `townOf(v.la,v.lo)`（用內嵌鄉鎮界 GeoJSON 點對多邊形算「縣市+區」）→ 否則 `v.city`。
- **分類篩選**：新增「動漫」按鈕 `data-c="動漫遊戲(ACG)"`（精確字串比對 `e.c`）。
- **開頁**：先用內嵌 `let DATA=...` 備援，再 `fetch('venues.json')`；頁尾標「（即時）」或「（內建備援）」。
- 內嵌備援 `DATA` 由 `update_all.py` 末段自動同步成最新 venues.json（故 file:// 雙擊也是最新）。
- **注意**：HTML 目前**直接手動編輯**（已不再用 build2.py）。改大段資料時用腳本替換 `let DATA = ...; let liveMode=false;` 那一行。

## 10. 如何擴充

### 新增官網爬蟲場館
- 結構乾淨者：編輯 `collect_venues.py` 的 `VENUES`（`key/city/lat/lng/url/list/path/exclude`，path 用 `auto` 或詳情連結特徵字串）。
- 結構特殊（民國年、特殊日期、需類別過濾）者：在 `collect_public.py` 加專屬 handler。
- 原則：結構混亂（講座/活動混列、無乾淨日期）的站先不要納入，以免汙染正確性。

### 新增使用者手動 ACG 活動
1. 使用者提供 Excel（欄位含活動名稱、起訖日、地點含地址、連結）。
2. 流程：**去重**（比對 venues.json 實際內容，重複跳過）→ **補資訊/定位**（地址→座標，缺則區級或列待確認，不杜撰）→ 寫入 `manual_extra.json` → 重跑 refresh → 同步內嵌備援。
3. `manual_extra.json` 格式同 `venue_extra.json`：`{"場館名":{"city","lat","lng","loc","url","ex":[{t,s,e,l,img}]}}`。

### 調整分類
改 `refresh_venues.py` 的 `CAT_TITLE`/`CAT_VENUE`（ACG 的 IP 名單也在 `CAT_TITLE` 第一條）。

### 前端加分類按鈕
在 HTML `#fcat` 加 `<button data-c="分類字串">標籤</button>`，字串需與後端分類完全一致。

## 11. 已知限制與設計決策（重要）

1. **定位精度**：exact / 區級 / 市級。目前 221 場館中約 161 為約略（多為文化部無座標→區級中心點）。其中約 **132 個「有地址、只是還沒精算座標」可批次升級成精確**——需可靠的地理編碼器（建議 TGOS 全國門牌或 Google Geocoding API，需金鑰）。Nominatim 免金鑰但對台灣門牌不準且易被擋。
2. **資料完整度受來源限制**：文化部以活動登錄為主；KV 圖稀少（多靠官網 og:image）；約半數展覽無圖、不少無連結。
3. **名稱**：絕大多數正常；少數需人工（如手動 ACG 的「澎湖/台東」應改為實際場館名）。
4. **分類為啟發式**：關鍵字推導；ACG 靠通用詞＋IP 名單，名單無法窮盡，新 IP 可能落到「其他」，補關鍵字即可。
5. **排除規則**：圖書館、郵政不收錄（已寫入管線）。
6. **沙箱網路限制**：自動化沙箱連不到多數 .gov.tw 與 C-LAB；爬蟲驗證需在使用者 Mac 上實測（排程也在 Mac 跑）。
7. **底圖/路名需連網**：file:// 離線可開（圖釘＋鄉鎮框線＋內嵌資料），但無街道圖磚與路名。
8. **著作權**：非池中（artemperor）僅用來「比對缺漏場館名稱」這個事實，座標自查 OSM、官網自找，不搬其彙整內容。

## 12. 驗證方式

- 資料管線：跑完看統計（場館/展覽/有圖/有連結）；抽查殘留（圖書館、郵政＝0）、過期＝0、無重複名稱。
- 前端：用 http server 或 file:// 開，確認底圖/路名載入、群集與圖釘、抽屜圖卡（標題/日期/**地址**/KV/連結）、各篩選計數（含「動漫」）、縮放時縣市↔區標籤切換、無 console 致命錯誤。
- 來源盤點：見 `場館來源清單.csv`（可篩選定位精度找出待精確化的場館）。
