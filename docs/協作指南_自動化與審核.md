## 三、每日自動更新 PR：怎麼審（河道 A）

1. **看 PR 的 `public/venues.json` diff**：綠色是今天新抓到的活動、紅色是消失的活動。
2. **新增的活動**：判斷是不是 ACG、資料對不對。
   - 要保留 → 什麼都不用做，直接合併 PR。
   - **不要**（非 ACG、重複、誤抓）→ 見第四節，把它記進 `review_decisions.json`，**在同一個 PR 裡一起改**，再合併。這樣下次不會又出現。
3. **消失的活動**：通常是展期結束被自動下架（正常）。若是「不該消失卻消失」（例如來源網站改版導致整批掉光），**先不要合併**，這是資料異常，要查來源。
4. **合併** → 觸發部署（部署設定完成後）→ 線上更新。

> 💡 想更省事：可以叫網頁版 cowork「幫我讀這個 PR，摘要今天新增/消失了哪些活動、哪些疑似非 ACG」，再依摘要決定。

## 四、怎麼「刪掉」一個活動讓它不再回來（河道 C）

自動爬蟲每天都會重抓，所以**光在 PR 裡把它刪掉沒用**，明天又會被抓回來。要把它記進決策檔：

編輯 `data/manual/review_decisions.json`，在 `rejected` 陣列加一筆：

```json
{
  "rejected": [
    {
      "key": "駁二藝術特區|某某非ACG特展|2026-08-01",
      "title": "某某非ACG特展",
      "venue": "駁二藝術特區",
      "reason": "純藝術展、非 ACG",
      "date": "2026-07-17"
    }
  ],
  "approved": []
}
```

- **`key` 是唯一有作用的欄位**，格式＝`場館名|正規化標題|開始日(YYYY-MM-DD)`，用 `|` 串接。
  - 正規化規則（`refresh_venues.py` 的 `stable_event_key()`）：去頭尾空白、全形空白→半形並壓縮、去除前後標點、「臺」→「台」。
  - 場館名要用 **venues.json 裡最終顯示的名稱**（改名/合併之後的名字）。
  - 拿不準 key 怎麼寫？在 repo 根目錄跑：
    ```bash
    python3 -c "import sys; sys.path.insert(0,'backend'); from refresh_venues import stable_event_key; print(stable_event_key('駁二藝術特區','某某非ACG特展','2026/08/01'))"
    ```
- `title` / `venue` / `reason` / `date` 只是給人看的備註，程式不讀。
- 之後每次輸出 `venues.json` 前，`refresh_venues.py` 會自動濾掉 `rejected` 名單裡的活動。**名單為空時完全不影響輸出**（no-op）。
- 反悔了？把該筆從 `rejected` 移除即可，活動下次自動回來。

> ⚠️ 決策是「你的最新意志」，改變心意直接改這個檔就好，不用糾結和舊紀錄矛盾。

## 五、手動新增內容：改哪個檔（河道 B）

**重要變更（2026/07/17）：手動活動的編輯入口，從 Excel 改成 `data/manual/acg_events.json`。**

- ✅ **新增/修改手動活動 → 編輯 `data/manual/acg_events.json`**（純文字、PR 看得懂 diff）。
- ⚠️ **不要再直接編輯 `data/manual/全台ACG活動.xlsx`**：管線現在讀 JSON，改 xlsx **不會生效**，只會讓兩邊不同步。xlsx 保留為歷史備份。
- `acg_events.json` 是原 Excel 的忠實文字鏡像（127 列 × 17 欄，已驗證兩者產出的 `manual_extra.json` **逐位元組相同**）。欄位與原 Excel 一致（`KV`、`類型 / Category`、`開始日期 / Start Date`…）；帶超連結的儲存格，其連結目標存在該列的 `_links` 物件裡。
- 想從目前 xlsx 重新產生一次鏡像（例如你堅持先在 Excel 編）：
  ```bash
  python3 backend/export_excel_to_json.py   # xlsx → data/manual/acg_events.json（覆蓋）
  ```
  但注意：**這會覆蓋 JSON 上的人工編輯**。所以請「二選一」：要嘛以 JSON 為準（推薦，別再動 xlsx）、要嘛以 xlsx 為準（每次編完重跑 export）。不要兩邊都改。
- 回退到舊行為（改回讀 xlsx）：刪掉 `data/manual/acg_events.json` 即可，importer 會自動退回讀 xlsx。

## 六、GitHub Actions workflows 一覽（`.github/workflows/`）

| 檔案 | 觸發 | 做什麼 |
|------|------|--------|
| `ci.yml` | 每次 push / PR | `py_compile` 全 backend + 驗證 `venues.json` 結構，**擋壞資料上線**。PR 紅燈就別合併。 |
| `check-sources.yml` | 手動（workflow_dispatch） | 實測雲端能否連到各場館網址（含花蓮文創 `nat.gov.tw`、圓山花博 `.taipei` 兩個政府網域）。**第一次啟用自動化前先手動跑一次**看結果。 |
| `daily-update.yml` | 每天 UTC 00:30（台灣 08:30）+ 手動 | 跑 `update_all.py` → 用 `peter-evans/create-pull-request` 開 PR，不直接 push main。 |

### 啟用步驟（第一次）
1. 到 GitHub → Actions → 手動跑一次 **check-sources**，看那 6+2 個網址回 200/3xx（連得到）還是 000/403/timeout（連不到）。
2. 若花蓮文創／圓山花博連不到 → 這兩館之後在 workflow 排除、改由本機補；其餘照樣每日自動跑。
3. 確認 `daily-update` 能成功開 PR 後，這條就自動化了。

## 七、還沒做、需要「有外網的環境」才能完成（待辦）

以下在雲端 CI 或你的 Mac（有外網）上執行，本專案的無外網開發環境做不了：

- **KV 圖在地化**：目前活動主視覺（`img`）多半直連 Instagram 圖床（`scontent.cdninstagram.com`，帶 `oe=` 到期參數），**會過期破圖**。需寫一步把圖下載到 `public/`（像 logo 那樣），改引用本地檔。容量很小（約 15–45MB）。
- **MapLibre 離線打包**：底圖函式庫仍走 unpkg CDN；unpkg 掛掉時圖釘/側欄仍可用，只是沒底圖（已容錯）。要完全離線需下載 maplibre 打包進站。
- **正式部署**：把 `public/` 上 GitHub Pages / Cloudflare Pages，取得正式網址後，**回頭把 HTML `<head>` 裡的 `og:url` / `canonical` 占位改成實際網址**，並設計一張 1200×630 的 `og:image` 後取消該行註解。

## 八、模型分工慣例（給 cowork 派工參考）

本輪採用的分工原則（可沿用）：
- **簡單、自成一檔的工作**（yaml/設定/樣板）→ 交給 **sonnet**。
- **一般複雜任務**（多檔、需理解架構）→ **opus**。
- **極度需要嚴謹邏輯 / 資料等值性證明**（如 Excel→JSON 遷移、去重、異常偵測）→ **fable**。
- 由 opus 當總協調：拆解、把「不可重疊的檔案範圍」分派給不同子代理並行、最後整合驗證與提交。
