最後更新：2026/06/27　撰寫：Claude（網頁維護員）
狀態：規格已與使用者確認方向（兩軸：主題 × 形式；兩層判斷）。標 ⚠️ 者為 AI 自行擬定、可由使用者/Claude Code 調整。

## 0. 需求追溯（為什麼改）
- 本地圖主打已收斂為 **ACG 展覽＋快閃店**，文化部開放資料只是背景層。
- 現行 7 類（視覺藝術/歷史人文/科學自然/工藝設計/親子兒童/動漫遊戲(ACG)/其他綜合）是博物館式主題分類，對 ACG 受眾幾乎無篩選力：使用者在乎的東西全擠進「動漫遊戲(ACG)」一格。
- 缺「形式」軸（展覽 vs 快閃店 vs 主題餐廳 vs 體驗）——這才是 ACG 受眾真正會用的篩選維度，且 Excel 已有「活動類別」欄在記。

## 1. 新分類：兩個正交軸

### 軸 A — 主題/性質（3 類）
| 值 | 說明 |
|---|---|
| `ACG` | 動漫、遊戲、卡通、角色 IP 授權 ← 主打 |
| `藝術設計` | 視覺藝術＋工藝設計合併 |
| `其他文化` | 歷史人文、科普自然、親子等文化部背景層全收 |

### 軸 B — 形式/型態（5 類）
| 值 | 說明 |
|---|---|
| `展覽` | 展覽／特展／原畫展／紀念展 |
| `快閃店` | 快閃店／期間限定店／主題店 |
| `主題餐廳` | 主題餐廳／咖啡廳／聯名餐飲 |
| `體驗活動` | 解謎、實境、主題日、音樂會、嘉年華、市集、戶外地景 |
| `其他` | 以上皆非 |

## 2. 資料模型變更（refresh_venues.py 輸出 venues.json）
- 每個活動物件 `ex[]`：
  - `c`：改為**軸 A 主題**（值：`ACG` / `藝術設計` / `其他文化`）。
  - 新增 `c2`：**軸 B 形式**（值：`展覽` / `快閃店` / `主題餐廳` / `體驗活動` / `其他`）。
  - 新增 `c_flag`（選用）：`"需人工確認"` 字串，當判斷不確定時帶上，前端可顯示提示、維護回報可彙整。
- 不要刪除既有欄位；前端會同步改。

## 3. 判斷演算法

### 軸 A（ACG 與否）— 兩層（使用者指定）
```
若 活動來源為 Excel/manual_extra：
    c = ACG                              # Excel 內一定是 ACG
否則（文化部等其他來源）：
    若 與任一 Excel 活動「重疊」：
        c = ACG                          # 繼承，不再猜
    否則：
        c = 關鍵字判斷(ACG / 藝術設計 / 其他文化)
```

**⚠️ 重疊（overlap）判定**：符合下列任一即視為重疊
1. 活動名稱正規化後相符（去空白/全半形/標點/書名號《》【】，可用 difflib 相似度 ≥ 0.85），或
2. IP 相符 **且** 場館相符 **且** 期間重疊（日期區間有交集）。

判不準（相似度落在 0.7–0.85 之間）→ `c` 先給關鍵字結果，並加 `c_flag="需人工確認"`，輸出到維護回報的待確認清單，不硬併。

### 軸 B（形式）
```
若 來源為 Excel：讀「活動類別 / Activity Category」欄 → 對應軸 B（見下表）；
    若 Excel 該欄為空或無法對應，再走關鍵字。
否則：關鍵字判斷。
```

**Excel「活動類別」→ 軸 B 對應（⚠️）**
| Excel 值（含變體） | 軸 B |
|---|---|
| 快閃店 / Pop Up / Pop-up Store | `快閃店` |
| 展覽 / Exhibition | `展覽` |
| 活動 / Event | `體驗活動` |
| 餐廳 / 咖啡 / Café / Cafe | `主題餐廳` |
| （空/其他）| 走關鍵字 |

> 註：Excel 另有「類型 / Category」欄常為「物販/Goods」，那是商品性質不是形式，**不要拿來當軸 B**。

## 4. ⚠️ 關鍵字表（請使用者/Claude Code 增刪）

軸 B（依序比對，先中先定；`展覽` 放最後當 fallback）：
- `快閃店`：`快閃`、`POP\s*-?\s*UP`(不分大小寫)、`期間限定`、`限定店`、`主題店`
- `主題餐廳`：`餐廳`、`咖啡`、`café`、`cafe`、`茶會`、`主題餐`
- `體驗活動`：`解謎`、`實境`、`主題日`、`音樂會`、`交響`、`嘉年華`、`市集`、`水舞`、`花火`；或（`聯名`/`×`）且場館屬球場/公園/戶外
- `展覽`（fallback）：`展`、`特展`、`原畫`、`紀念展`、`Exhibition`

軸 A 關鍵字（**僅用於與 Excel 無重疊的非 Excel 資料**）：
- `ACG`：`動畫`、`動漫`、`漫畫`、`聲優`、`Vtuber`、`遊戲`、`角色`、`IP`、常見 IP 名（航海王、鬼滅、咒術迴戰、三麗鷗、Chiikawa、寶可夢…可維護一張 IP 名單）
- `藝術設計`：`美術`、`畫展`、`攝影`、`設計`、`工藝`、`陶`、`書法`、`雕塑`
- 其餘 → `其他文化`

**優先序（全域）**：Excel 欄位 ＞ 標題關鍵字 ＞ 場館型態關鍵字 ＞ 預設。
**硬規則**：形式不得被場館型態蓋過（百貨/文創園區裡的快閃店＝`快閃店`，不是場館類型）。

## 5. 前端（taiwan-exhibition-map.html）
- 把現有「分類」單一篩選改成**兩個篩選器**：
  - 「主題」：ACG / 藝術設計 / 其他文化（預設可全選，或預設聚焦 ACG）。

- `c_flag="需人工確認"` 的活動：照常顯示，可加一個小提示圖示（非必要）。
